Content¶

In this notebook, we show simple example usages of the two ensemble classifiers we have implemented, namely:

  1. Weighted Ensemble Classifier: the weighted ensemble classifier proposed in [1] (ensemble.py)
  2. Cost-Sensitive Weighted Ensemble Classifier: the extension that handles cost-sensitive applications (costsensiv_ensemble.py)

We also compare the performance of our implementation with other classifiers available in scikit-multiflow.

References¶

[1] "Mining Concept-Drifting Data Streams using Ensemble Classifier", by Haixun WANG, Wei FAN, Philip S. YU, Jiawei HAN.

C.1 - Weighted Ensemble Classifier¶

In [6]:
# instantiate a classifier
clf = WeightedEnsembleClassifier(K=10, base_learner=HoeffdingTree(), S=250, cv=5)

# some dudes to compare with
h = [
    clf, 
    HoeffdingTree(), 
    OzaBagging(base_estimator=HoeffdingTree())
    ]

# prepare the evaluator
evaluator = EvaluatePrequential(pretrain_size=1000, max_samples=100000, show_plot=True,
                                metrics=['accuracy', 'kappa'], batch_size=10)

# run
stream.prepare_for_use()
evaluator.evaluate(stream=stream, model=h, model_names=["AWE", "Hoeffding", "OzaBag"])

C.2 - Cost-Sensitive Weighted Ensemble Classifier¶

Since the usage of the cost-sensitive extension is quite specific, we do not compare it with any other classifer. We let it run alone and show its performance in terms of accuracy and kappa statistics.

In [7]:
# prepare the stream
stream = FileStream('data/creditcard.csv', n_targets=1, target_idx=-1)
stream.prepare_for_use()

# create the cost-sensitive classifier
clf = CostSensitiveWeightedEnsembleClassifier(base_learner=HoeffdingTree(), K=250, cv=None)

# setup the evaluator
evaluator = EvaluatePrequential(pretrain_size=1000, max_samples=20000, show_plot=True,
                                metrics=['accuracy', 'kappa'], batch_size=10)

# 4. Run
evaluator.evaluate(stream=stream, model=clf)
